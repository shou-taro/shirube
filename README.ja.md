## AI が SQL を書く時代に

自分で SQL を書く機会は、以前より減っているはずです。多くは AI が書いてくれます。それでも、その SQL が動く相手は結局あなたのスキーマであり、スキーマそのものを理解する仕事は人に残ります。AI にうまく指示を出すにも、返ってきた結果を確かめるにも、データが実際どこにあるのかを把握しておく必要があるからです。かつては自分でクエリを書きながら自然と身についていたその理解が、いまは得にくくなっています。

shirube は、そのスキーマ理解を助けるツールです。起動するとインタラクティブな ER 図が開き、データベースを地図のように探索できます。テーブルを検索して選び、そこからリレーションをたどって周辺へ広げていく。DDL を読んだりクエリを書いたりしなくても、全体のつながりがひと目で分かります。

目指しているのは「SQL を書かせないこと」ではなく、「データベースの中で迷子にならないこと」です。たとえば、次のような疑問にすぐ答えられます。

- このデータはどこにあり、この列はどのテーブルのものか
- 2 つのテーブルはどう関係し、この外部キーはどこへつながるのか
- そもそも、どのテーブルから見ればよいのか

もちろん従来どおり、ドキュメントの乏しい数百テーブルのプロジェクトに入ったとき、素早く全体像をつかむのにも役立ちます。shirube は SQL IDE でもデータベース管理ツールでもなく、データベースを理解するためのツールです。

## 主な機能

現在のベータ版で、次の機能がすべて使えます。

- **ER 図がホーム画面。** shirube が ER 図を自動生成し、最も多くつながっているテーブルを中心に表示します。一度に映すのはそのテーブルと直接の隣接だけなので、数百のテーブルに埋もれることなく、1 つずつ隣へたどっていけます。
- **テーブルの詳細。** 各列の型・主キー・NULL 可否に加えて、「参照している / 参照されている」に分けたリレーションを表示します（ビューが参照するテーブルも含みます）。
- **リレーションをたどる。** 関連テーブルをクリックすると、地図がそこへ滑らかに移動します。そのままつながりをたどり続けられます。
- **データプレビュー。** テーブルやビューの実データを、ER 図の下のパネルで確認できます。列見出しのクリックで並べ替え、簡単な絞り込み、ページ送りに対応しています。
- **すばやい検索。** <kbd>⌘K</kbd> / <kbd>Ctrl K</kbd> で、任意のテーブルや列にすぐ移動できます。
- **接続の保存。** 複数の PostgreSQL 接続に名前を付けて管理できます。パスワードは設定ファイルではなく、OS のキーチェーンに保存されます。
- **ライト / ダークテーマ。**

## 安全性への配慮

shirube は「操作していて怖くない」ことを大切にしています。

- **読み取り専用。** すべての接続を、ステートメントタイムアウト付きの読み取り専用モードで開きます。書き込みやスキーマ変更を行うことはありません。
- **ローカル完結。** 実行は手元のマシンの中だけで完結し、待ち受けは `127.0.0.1` に限定されます。接続情報やデータが外部に送られることはなく、パスワードは OS のキーチェーンに保管されます。
- **ログはメタデータのみ。** ローカルのログには、問題の切り分けに必要な情報（エラーやリクエストの所要時間）だけを記録します。データの中身が書き込まれることはありません。

## 使い方

**必要なもの:** 接続できる PostgreSQL データベースと、[uv](https://docs.astral.sh/uv/)（`uvx` が使えるようになります）。

```bash
uvx shirube
```

ローカルサーバーが起動し、ブラウザで shirube が開きます。PostgreSQL への接続を追加すれば、すぐに探索を始められます。接続先はローカルでもリモートでも構いません。

> shirube は、渡された認証情報でそのまま接続します。必要なのは `CONNECT` と `SELECT` の権限を持つ読み取り専用ロールだけで、それ以上の権限は与えないことをおすすめします。

### サンプルデータベースで試す

手元に試せる PostgreSQL が無い場合は、サンプルデータベース（[pagila](https://github.com/devrimgunduz/pagila)）で動きを確かめられます。**Docker** と、このリポジトリのクローンが必要です。

```bash
git clone https://github.com/shou-taro/shirube.git
cd shirube
./scripts/dev-db.sh up
```

`postgresql://postgres:postgres@127.0.0.1:5432/pagila` が起動します。shirube でこの接続を追加すれば、そのまま探索を始められます。

## ロードマップ

shirube の開発は 3 つの段階で進みます。

- **いま — Explore（ベータ）。** 上記の ER 図・テーブル詳細・リレーション・データプレビュー・検索。
- **次 — AI navigator。** データの在りかやテーブルの関係を、自然な言葉で尋ねられるようにします。AI はあくまで案内役（ナビゲータ）であり、SQL を生成したり、データを変更したりはしません。これがまだ実装していない目玉機能です。
- **その先 — Analyse & Manage。** より高度な GUI での絞り込みや集計、ビューの保存、AI によるリレーションの提案、意味ベースの検索。さらに、安全な GUI 編集やチーム / セルフホスト機能へと広げていきます。対応データベースは PostgreSQL に続いて MySQL・SQL Server を予定しています。
