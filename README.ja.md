## AI コーディング時代のために

昔ほど自分で SQL を書かなくなりました——多くは AI が書いてくれます。けれど、その SQL が走る先は**あなたのスキーマ**であり、スキーマを理解する仕事は依然として人に残ります。AI にうまく指示するにも、返ってきたものを確かめるにも、データが実際どこにあるのかを考えるにも。かつては自分でクエリを書く過程で自然と身についた理解が、もう手に入りません。

shirube は、その理解が宿る場所です。起動するとインタラクティブな ER 図が開き、データベースを地図のように探索できます——テーブルを検索し、そこに焦点を当て、リレーションを外へと辿る。DDL を読むことも、クエリを 1 行書くこともなく、全体のつながりが見えてきます。

目的は「SQL を書かないこと」ではありません。**「迷子にならないこと」**です。shirube は、こんな問いに答えます：

- このデータはどこにある？ この列はどのテーブルのもの？
- この 2 つのテーブルはどう関係している？ この外部キーはどこへ続く？
- そもそも、どのテーブルから見ればいい？

従来の使い方——ドキュメントのない数百テーブルのプロジェクトに放り込まれ、素早く足場を掴みたいとき——にも、同じく役立ちます。shirube は SQL IDE でも DB 管理コンソールでもありません。データベースを**理解する**ためのツールです。

## 機能

以下はすべて、ベータの現時点で動作します：

- **ER 図がホーム画面。** shirube が図を自動生成し、最も多く接続されたテーブルを中心に据えます。あるテーブルとその直近の隣接だけを表示し（数百の壁ではなく）、1 ホップずつ外へ辿ります。
- **テーブル詳細。** 型・主キー・NULL 可否を備えた列一覧に加え、*参照している / 参照されている* に分けたリレーション（ビューが読むテーブルも含む）。
- **リレーション・ナビゲーション。** 関連テーブルをクリックすると地図がそこへ滑るように移動し、つながりを辿り続けられます。
- **データプレビュー。** テーブルやビューの実際の行を、地図の下のドロワーで閲覧。列クリックでソート、簡単な列フィルタ、ページング付き。
- **即時検索。** <kbd>⌘K</kbd> / <kbd>Ctrl K</kbd> で任意のテーブル・列へ一足飛び。
- **保存済み接続。** 複数の PostgreSQL プロファイルを管理。パスワードは設定ファイルではなく OS のキーチェーンに保管します。
- **ライト / ダークテーマ。**

## 安全設計

shirube は、決して危険に感じさせないことを目指しています。

- **読み取り専用。** すべての接続は statement timeout 付きの読み取り専用で開きます。shirube がデータベースを変更することはありません——書き込みもスキーマ変更も、一切ありません。
- **ローカルファースト。** あなたのマシン上で動き、`127.0.0.1` のみにバインドします。データベースの認証情報もデータもマシンの外に出ず、パスワードは OS キーチェーンに保管されます。
- **メタデータのみのログ。** ローカルログは問題の診断に必要な情報（エラーやリクエスト時間）を記録しますが、データの値は決して記録しません。

## セットアップ

**必要なもの：** 到達可能な PostgreSQL データベースと、[uv](https://docs.astral.sh/uv/)（`uvx` を提供します）。

```bash
uvx shirube
```

ローカルサーバが起動し、ブラウザで shirube が開きます。PostgreSQL への接続を追加すれば探索開始です——データベースはローカルでもリモートでも構いません。

> shirube は与えられた認証情報でそのまま接続します。`CONNECT` と `SELECT` を持つ読み取り専用ロールがあれば十分で、それ以上の権限は持たせないのが理想です。

## ロードマップ

shirube の開発は 3 つのフェーズで進みます。

- **いま — Explore（ベータ）。** 上記の ER 図・テーブル詳細・リレーション・データプレビュー・検索。
- **次 — AI navigator。** データの在り処やテーブルのつながりを自然な言葉で尋ね、案内役に導いてもらう。AI は SQL ジェネレータではなく *ナビゲータ* で、何も変更しません。これがまだ残る目玉機能です。
- **その先 — Analyse & Manage。** より高機能な GUI フィルタと集計、保存ビュー、AI によるリレーション提案、セマンティック検索。続いて安全な GUI 編集、チーム / セルフホスト機能。MySQL・SQL Server は PostgreSQL に続きます。
